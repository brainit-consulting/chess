Scorpion Chess — Git Cheat Sheet

Manual workflow used in our repo (Windows + VS Code + worktrees). Includes the v1.1.67 release example.
Goal: keep main clean & ship tagged releases

1) Sanity checks (run constantly)

fast confidence
Where am I? (branch + cleanliness)
git branch --show-current
git status
What version am I on? (tag proximity)
git describe --tags
git describe --tags --exact-match
✓
Example: v1.1.66-1-g9756f67 means 1 commit after v1.1.66 (pre v1.1.67).

2) Worktrees: why VS Code “can’t switch to main” sometimes

worktree rule
Rule: a branch can only be checked out in one worktree at a time.
  • If main is checked out in another folder, Git will block switching to it here.
  • Fix by switching folders, or “freeing” main from the other worktree.
Free main from another worktree (example: chess-baseline)
cd /d H:\chess-baseline
git switch -c parking/baseline
Then switch this folder to main
cd /d H:\chess
git switch main
git pull origin main

3) Archive WIP without committing (safe, reversible)

keep main clean
Stash everything (tracked + untracked)
git stash push -u -m "WIP: king-safety experiments"
git status
  • Use when you want a clean main immediately.
  • LF/CRLF warnings on Windows are normal.
Restore later
git stash list
git stash show -p stash@{0}
git stash pop
Optional: permanent archive marker (tag)
git tag archive/v1.1.67-king-safety
git push origin archive/v1.1.67-king-safety

4) Release flow (example: ship v1.1.67)

minimal diff
A) Update UI footer version (Scorpion Chess)
git grep -n "APP_VERSION" src
# edit: src/ui/ui.ts
# const APP_VERSION = 'v1.1.67';
B) Update CHANGELOG
# edit: CHANGELOG.md (top)
## v1.1.67
- Fix: Analyzer links now open via anchor navigation for Edge compatibility.
C) Review + commit only what you intend
git diff -- src/ui/ui.ts
git diff -- CHANGELOG.md
git status --porcelain

git add src/ui/ui.ts CHANGELOG.md
git commit -m "chore(release): v1.1.67"
D) Tag + push
git tag v1.1.67
git push origin main
git push origin v1.1.67
✓
Verify release:
git describe --tags --exact-match
# expect: v1.1.67

5) VS Code: open the right folder every time

avoid confusion
Open main in H:\chess
cd /d H:\chess
code .
  • Branch shown bottom-left should be main.
  • Confirm via git branch --show-current.
Quick keyboard tip
  • Ctrl + Shift + P → “Git: Checkout to…”
  • If worktrees block it, switch folders (not branches).
